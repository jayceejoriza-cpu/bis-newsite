# Testing Guide: Password-Protected Database Backup

## Prerequisites
- Ensure you have a user account in the system
- Know your login credentials (username and password)
- Have access to the backup.php page

## Test Scenarios

### Test 1: Successful Backup with Correct Password ✅

**Steps:**
1. Log in to the system
2. Navigate to the Database Backup page (backup.php)
3. Click the "Generate & Download Backup" button
4. A modal should appear asking for your password
5. Enter your correct password
6. Click "Confirm & Backup"

**Expected Results:**
- ✅ Backup file downloads automatically
- ✅ Filename format: `bmis_backup_YYYY-MM-DD_HH-MM-SS_by_username.sql`
- ✅ File contains SQL backup with header showing your username
- ✅ No error messages displayed
- ✅ Entry created in audit_logs table with action 'BACKUP'

---

### Test 6: Verify Audit Logs

**Steps:**
1. Perform Test 1 (successful backup)
2. Perform Test 2 (failed backup with wrong password)
3. Check the audit_logs table in the database

**SQL Query:**
```sql
SELECT * FROM audit_logs 
WHERE action IN ('BACKUP', 'BACKUP_AUTH_FAILED', 'BACKUP_FAILED')
ORDER BY created_at DESC 
LIMIT 10;
```

**Expected Results:**
- ✅ Entry for successful backup with action 'BACKUP'
- ✅ Entry for failed authentication with action 'BACKUP_AUTH_FAILED'
- ✅ Each entry includes:
  - user_id (your user ID)
  - action type
  - timestamp (created_at)
  - ip_address
  - user_agent
  - new_values (JSON with additional info)

## Database Verification Queries

### Check Recent Backups
```sql
SELECT 
    al.id,
    al.user_id,
    u.username,
    u.full_name,
    al.action,
    al.ip_address,
    al.created_at,
    al.new_values
FROM audit_logs al
LEFT JOIN users u ON al.user_id = u.id
WHERE al.action IN ('BACKUP', 'BACKUP_AUTH_FAILED', 'BACKUP_FAILED')
ORDER BY al.created_at DESC
LIMIT 20;
```

### Count Backup Activities by User
```sql
SELECT 
    u.username,
    u.full_name,
    COUNT(*) as total_backups,
    SUM(CASE WHEN al.action = 'BACKUP' THEN 1 ELSE 0 END) as successful,
    SUM(CASE WHEN al.action = 'BACKUP_AUTH_FAILED' THEN 1 ELSE 0 END) as auth_failed,
    SUM(CASE WHEN al.action = 'BACKUP_FAILED' THEN 1 ELSE 0 END) as technical_failed
FROM audit_logs al
LEFT JOIN users u ON al.user_id = u.id
WHERE al.action IN ('BACKUP', 'BACKUP_AUTH_FAILED', 'BACKUP_FAILED')
GROUP BY u.username, u.full_name
ORDER BY total_backups DESC;
```

## Troubleshooting

### Issue: Modal doesn't appear
**Solution:** Check browser console for JavaScript errors

### Issue: Password verification fails even with correct password
**Solution:** 
- Verify user exists in database
- Check password hash in database
- Ensure session variables are set correctly

### Issue: Audit logs not being created
**Solution:**
- Verify audit_logs table exists
- Check database connection
- Review PHP error logs

### Issue: Backup file not downloading
**Solution:**
- Check PHP output buffering settings
- Verify file permissions
- Check browser download settings
